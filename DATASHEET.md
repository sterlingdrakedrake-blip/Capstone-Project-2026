# Datasheet for Dataset: BBO Capstone Query Log

This datasheet follows the Datasheets for Datasets framework (Gebru et al., 2021) and documents the dataset accumulated during the Black-Box Optimization Capstone project.

---

## Motivation

**For what purpose was the dataset created?**
The dataset was created to support Bayesian optimization of eight unknown black-box functions as part of a taught capstone project. Each function simulates a real-world optimization problem (radiation detection, drug discovery, warehouse placement, etc.) where function evaluations are expensive and limited.

**Who created the dataset and on whose behalf?**
The dataset was created by the project participant through iterative weekly query submissions to a course portal. The portal evaluates submitted input vectors against hidden functions and returns scalar output values.

**Was there any funding associated with the creation of this dataset?**
The dataset was created as part of a graded academic assessment. No external funding was involved.

---

## Composition

**What do the instances represent?**
Each instance is a (input vector, output scalar) pair representing one evaluation of a black-box function. The input vector contains continuous values in [0, 1] and the output is a single real-valued scalar.

**How many instances are there in total?**
The complete final dataset (all 13 query rounds) contains:

| Function | Dims | Initial Points | Query Rounds | Total Points |
|---|---|---|---|---|
| F1 | 2 | 10 | 13 | 23 |
| F2 | 2 | 10 | 13 | 23 |
| F3 | 3 | 15 | 13 | 28 |
| F4 | 4 | 30 | 13 | 43 |
| F5 | 4 | 20 | 13 | 33 |
| F6 | 5 | 20 | 13 | 33 |
| F7 | 6 | 30 | 13 | 43 |
| F8 | 8 | 40 | 13 | 53 |

**Does the dataset contain all possible instances or is it a sample?**
It is a sample. The input space is the continuous unit hypercube [0, 1]^d for each function. The dataset represents a small, strategically selected subset.

**Is there a label or target associated with each instance?**
Yes. Each input vector has an associated scalar output value returned by the course portal. All tasks are maximization problems.

**Are there any known errors, sources of noise, or redundancies?**
Some functions (particularly F2 and F6) appear to have noisy outputs, where the same input region queried across rounds returns inconsistent values. Function 1 returns values indistinguishable from zero across all 23 observations (13 query rounds plus 10 initial points), which may reflect a very narrow isolated peak whose basin of attraction is smaller than the spacing between any two sampled points.

**Does the dataset contain data that might be considered confidential?**
No. All inputs were generated by the participant and outputs are synthetic scalars with no personal or sensitive information.

---

## Collection Process

**How was the data associated with each instance acquired?**
Initial data points were provided by the course at the start of the project as .npy files. All subsequent points were generated by submitting query inputs to a course portal, which returned the corresponding output value.

**What mechanisms or procedures were used to collect the data?**
Rounds 1 through 3 used Gaussian Process regression with UCB acquisition and a uniform kappa of 2.576. Rounds 4 onward used adaptive, function-specific kappa values in the range {0.5, 1.0, 1.5, 2.0, 2.5} based on observed performance. Function 1 used GP-UCB for Round 1 only, then switched to a dynamic max-distance grid sweep (Rounds 2-9), then to a pre-planned 4-point systematic sweep (Rounds 10-13) after no signal was detected. No positive output was ever returned.

**Over what timeframe was the data collected?**
One query per function per week over a 13-week academic term.

**Were any ethical review processes conducted?**
Not applicable. The dataset contains no personal data and was generated in a controlled academic simulation environment.

---

## Preprocessing and Cleaning

**Was any preprocessing/cleaning/labeling of the data done?**
Minimal preprocessing was applied:
- All query values were clamped to [0.0, 0.999999] to satisfy portal format constraints.
- All values were formatted to six decimal places.
- Output values returned in Python float notation (e.g. `np.float64(...)`) were parsed to plain floats.

**Was the raw data saved in addition to the preprocessed/cleaned data?**
Yes. Raw output text files from the portal are stored in the `outputs/` folder of the repository.

---

## Uses

**Has the dataset been used for any tasks already?**
The dataset has been used exclusively for the BBO capstone optimization task: fitting GP surrogate models and selecting query points via UCB acquisition.

**What other tasks could the dataset potentially be used for?**
- Benchmarking surrogate model accuracy on small datasets.
- Comparing acquisition function strategies (UCB vs EI vs PI) retrospectively.
- Demonstrating the curse of dimensionality in black-box optimization.
- Teaching material for Bayesian optimization courses.

**Is there anything about the composition of the dataset that might impact future uses?**
The dataset is small (23 to 53 points per function). Any model trained on it will be data-limited and results should not be generalized beyond the specific functions represented. For higher-dimensional functions (F7 6D, F8 8D), even 43-53 observations provide sparse coverage of the input hypercube.
